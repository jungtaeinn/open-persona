# 기능 QA 테스트 가이드

## 테스트 유형
- 스모크 테스트: 핵심 기능이 작동하는지 빠르게 확인
- 회귀 테스트: 새 변경이 기존 기능을 깨뜨리지 않는지 확인
- 경계값 분석: 입력의 최소/최대/경계에서 동작 확인
- 탐색적 테스트: 시나리오 없이 직감적으로 문제 탐색

## 기능 테스트 체크리스트
- 정상 시나리오 (Happy Path): 기대대로 동작하는가?
- 비정상 시나리오 (Sad Path): 잘못된 입력에 적절히 대응하는가?
- 경계값: 빈 입력, 최대 길이, 특수문자, 유니코드
- 권한: 로그인/비로그인, 관리자/일반 사용자
- 동시성: 여러 탭/창에서 동시 조작

## 웹 애플리케이션 테스트 포인트
- 폼 유효성: 필수값, 형식 검증, 에러 메시지
- 네비게이션: 라우팅, 뒤로가기, 딥링크
- 상태 관리: 새로고침 후 상태 유지
- API 에러 처리: 네트워크 끊김, 타임아웃, 4xx/5xx
- 파일 업로드: 파일 크기, 형식, 다중 업로드

## 크로스 브라우저 테스트
- Chrome, Firefox, Safari, Edge 최신 버전
- iOS Safari, Android Chrome 모바일 브라우저
- 브라우저별 CSS 차이 확인
- JavaScript API 호환성

## 성능 테스트 포인트
- 페이지 로드: First Contentful Paint < 1.5s
- 인터랙션: Interaction to Next Paint < 200ms
- 레이아웃 안정성: Cumulative Layout Shift < 0.1
- 대량 데이터: 리스트 1000건 이상 스크롤 성능

## 버그 리포트 작성법
- 제목: [영역] 간결한 문제 설명
- 재현 순서: 1. 단계 → 2. 단계 → 3. 단계
- 기대 결과 vs 실제 결과
- 환경: OS, 브라우저, 버전
- 스크린샷/영상 첨부
- 심각도: Critical / Major / Minor / Trivial
