# 디자인 QA 체크리스트

## 레이아웃 검증
- Figma/디자인 시안과 실제 구현 간격(padding, margin) 일치 여부
- 그리드 시스템 준수: 12컬럼 그리드, 거터 간격
- 정렬: 좌/중앙/우 정렬 일관성
- 반응형 브레이크포인트별 레이아웃 확인 (320, 768, 1024, 1440px)

## 타이포그래피
- 폰트 패밀리, 사이즈, weight 일치
- 줄간격(line-height), 자간(letter-spacing) 확인
- 텍스트 오버플로 처리: ellipsis, 줄바꿈, 말줄임표
- 최대/최소 글자 수 시나리오 테스트

## 색상 및 테마
- 디자인 토큰 색상값 정확히 매칭
- 다크 모드 / 라이트 모드 전환 확인
- 색상 대비 WCAG 2.1 AA 기준 (4.5:1 이상)
- 호버/포커스/액티브 상태 색상

## 컴포넌트 상태
- Default, Hover, Active, Focus, Disabled 상태 구현
- 로딩 상태: 스켈레톤 UI 또는 스피너
- 빈 상태 (Empty State): 적절한 안내 메시지
- 에러 상태: 에러 메시지 표시 위치, 스타일

## 아이콘 및 이미지
- 아이콘 사이즈, 색상 일치
- SVG vs PNG 적절한 사용
- 이미지 비율 유지 (object-fit)
- alt 텍스트 제공 (접근성)
- 레티나 디스플레이 대응 (@2x)

## 간격 시스템
- 4px 또는 8px 기반 간격 시스템 준수
- 컴포넌트 간 간격 일관성
- 섹션 간 여백 통일

## 애니메이션 / 트랜지션
- 디자인 시안의 모션 명세 반영
- 트랜지션 duration, easing 확인
- 과도한 애니메이션 방지 (prefers-reduced-motion 대응)

## 접근성 (a11y)
- 키보드 탐색 가능 여부 (Tab, Enter, Escape)
- 포커스 표시 (focus-visible)
- 스크린 리더 호환 (aria-label, role)
- 색각 이상자 대응 (색상 외 구분 수단 제공)
