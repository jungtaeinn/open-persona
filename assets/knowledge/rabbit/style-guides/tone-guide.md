# 번역 스타일 가이드

## 톤 & 보이스 설정

### 공식 문서 번역
- 격식체(합니다) 사용
- 수동태/명사형 문장 적절히 활용
- "~하시기 바랍니다", "~참조하십시오"

### 기술 문서 번역
- 해요체 또는 합니다체
- 코드/명령어는 원문 그대로 유지
- 기술 용어: 공인된 한국어 표기 우선, 없으면 원문 병기
- 예: "API 엔드포인트(endpoint)를 설정합니다"

### 마케팅/UI 텍스트
- 간결하고 행동 지향적
- 버튼: 동사형 ("시작하기", "다운로드", "자세히 보기")
- 알림: 짧고 명확 ("저장되었습니다", "오류가 발생했습니다")
- CTA: "지금 시작해보세요", "무료로 체험하기"

### 대화/캐주얼 번역
- 해체 또는 해요체
- 구어체 표현 자연스럽게 활용
- 이모지 적절히 활용

## 공통 원칙
1. 원문의 의미와 뉘앙스를 최대한 보존
2. 대상 언어(타겟)의 자연스러운 문장 구조 우선
3. 전문 용어는 일관되게 통일
4. 불필요한 번역투 제거
5. 문화적 맥락 고려 (날짜 형식, 숫자 표기 등)
6. 이름/브랜드는 원문 유지 또는 공식 표기 사용

## 숫자/단위 표기
- 한국: 쉼표(,) = 천 단위 구분, 마침표(.) = 소수점
- 날짜: YYYY년 M월 D일 / YYYY-MM-DD
- 시간: 오전/오후 H시 M분 또는 24시간제
- 화폐: ₩ (원), $ (달러), ¥ (엔)
