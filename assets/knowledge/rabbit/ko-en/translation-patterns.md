# 한국어 → 영어 번역 패턴 가이드

## 어순 변환 원칙
- 한국어 SOV → 영어 SVO: 주어 + 동사 + 목적어
- 수식어 위치: 한국어는 앞에서 수식, 영어는 뒤에서 수식하는 경우 많음
- "~하는 것" → gerund (-ing) 또는 to-infinitive
- "~에 대해" → about / regarding / concerning
- "~함에 따라" → as / in accordance with

## 조사별 영어 전치사 매핑
- ~에 (장소): at / in / on
- ~에 (시간): at (시각), in (월/년), on (날짜/요일)
- ~에서: from / at / in
- ~(으)로: to / by / with / as
- ~의: of / 's (소유격)
- ~와/과: and / with
- ~보다: than / more than
- ~처럼: like / as

## 존대/격식 변환
- 격식체 (합니다): formal English — "We would like to inform you that..."
- 일반 격식 (해요): standard — "Please note that..."
- 비격식 (해): casual — "Just so you know..."
- 수동태 선호: 한국어 주어 생략 시 영어는 수동태 활용

## IT/기술 용어 번역 관례
- 개발 환경 → development environment
- 배포 → deployment / release
- 운영 환경 → production environment
- 형상 관리 → version control
- 이슈 → issue / bug / ticket
- 요구사항 → requirements
- 기획서 → product specification / PRD
- 명세서 → specification document

## 관용 표현 및 자연스러운 전환
- "검토 부탁드립니다" → "Please review this at your earliest convenience"
- "확인 부탁드립니다" → "Could you please confirm..."
- "~건에 대해" → "Regarding [topic]..."
- "첨부 파일 참조 바랍니다" → "Please refer to the attached file"
- "~에 문의 주세요" → "Please contact [team/person] for..."
- "진행 상황 공유드립니다" → "Here's an update on the progress"

## 번역 품질 체크포인트
- 직역 vs 의역: 자연스러운 영어 문장이 되는지
- 주어 명시: 한국어에서 생략된 주어를 영어에서 보충
- 시제 일관성: 과거/현재/미래 혼용 주의
- 관사: a/an/the 적절한 사용 (한국어에 없는 개념)
- 단수/복수: 영어의 수 일치 확인
