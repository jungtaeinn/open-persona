# 영어 → 한국어 번역 패턴 가이드

## 어순 변환 원칙
- 영어 SVO → 한국어 SOV: 주어 + 목적어 + 동사
- 관계절: 영어 뒤에서 수식 → 한국어 앞에서 수식 ("the man who..." → "...하는 사람")
- 분사구문: 영어 ~ing → 한국어 "~하면서", "~한 채로"
- 접속사: because → ~때문에, although → ~에도 불구하고

## 관사/복수 처리
- a/an → 보통 생략 또는 "하나의", "어떤"
- the → 문맥에 따라 "그", "해당", 또는 생략
- 복수 → 한국어에서 복수 접미사(-들) 필요 시에만

## 자연스러운 한국어 문장 만들기
- 번역투 회피: "그것은 ~이다" → "~입니다"
- 주어 과잉 제거: 한국어는 주어 생략이 자연스러움
- 긴 문장 분리: 영어 한 문장 → 한국어 2~3문장으로 분리
- 능동태 선호: 영어 수동태 → 한국어 능동태로 전환

## IT/기술 용어 한국어 표기
- deployment → 배포
- migration → 마이그레이션
- refactoring → 리팩토링
- dependency → 의존성
- endpoint → 엔드포인트
- middleware → 미들웨어
- authentication → 인증, authorization → 인가
- repository → 리포지토리 / 저장소
- pull request → 풀 리퀘스트 / PR
- code review → 코드 리뷰

## 격식 수준 가이드
- Formal → 합니다체: "~합니다", "~습니다"
- Semi-formal → 해요체: "~해요", "~어요"
- Casual → 해체: "~해", "~야"
- 비즈니스: "~드립니다", "~바랍니다", "검토 부탁드립니다"

## 번역 품질 체크포인트
- 번역투 여부: "그것은", "나는" 과다 사용
- 조사 자연스러움: ~은/는, ~이/가, ~을/를 적절한 사용
- 문장 호흡: 한 문장이 너무 길지 않은지
- 맥락 보존: 원문의 뉘앙스가 살아있는지
- 용어 일관성: 같은 원어를 같은 한국어로 통일
